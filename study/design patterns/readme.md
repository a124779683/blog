#  设计模式学习Summary
设计模式的一个简要回顾，对每一种设计模式的学习在本文末尾进行收录
## 
1. 设计模式的概念和目的  
设计模式是为了解决特定环境下的一些问题而产生的**常用**的解决方案。  
使用设计模式的目的我认为是为了使我们的程序更利于维护和扩展，易于阅读，代码健壮性高，另外代码重用使得我们开发成本得以降低。

2. 设计模式的来历  
GoF(Gang of Four)四人帮编写的书籍《设计模式》的问世影响非常深远，里面收编了**23个常用的设计模式**。  

3. 设计模式的四个基本要素  
 + 模式名称
 + 问题
 + 解决方案
 * 效果
